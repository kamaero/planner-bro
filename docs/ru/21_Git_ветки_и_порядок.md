# PlannerBro — Git ветки и порядок работы

Обновлено: 2026-03-19

Этот документ нужен, чтобы не путаться между ветками и не устраивать "квест на выживание" перед каждым `git switch`.

## 1. Роли веток

- `main`:
  - стабильная базовая линия;
  - сюда не ведем повседневную тяжелую распиловку.
- `codex/temp-assignees-and-ai-bulk`:
  - активная рабочая ветка текущего цикла разгрузки/контрактов;
  - сюда идут наши ежедневные коммиты по текущему фронту работ.
- `codex/hotfix-date-assignee-deps-20260304`:
  - отдельная историческая hotfix-ветка на origin;
  - не использовать как рабочую по умолчанию.

## 2. Единое правило на каждый день

Работаем в двух дорожках:

- дорожка чтения/сверки: `main`;
- дорожка реализации: `codex/temp-assignees-and-ai-bulk`.

То есть: сверились с `main`, вернулись в рабочую, продолжили пилить.

## 3. Стандартный цикл (без магии)

1. Обновить refs:
   - `git fetch --prune origin`
2. Подтянуть базу:
   - `git switch main`
   - `git pull --ff-only origin main`
3. Вернуться в рабочую:
   - `git switch codex/temp-assignees-and-ai-bulk`
   - `git pull --ff-only origin codex/temp-assignees-and-ai-bulk`
4. Делать задачу, коммитить маленькими порциями.
5. Пушить сразу после логически законченного шага.

## 4. Антипаника-чек перед переключением

Перед `switch` всегда проверить:

- `git status --short --branch`

Если дерево грязное:

- либо честно коммитим;
- либо осознанно stash;
- либо явно отменяем только свои локальные временные правки.

Случай "ой, а куда делось" обычно начинается с игнора этого пункта.

## 5. Быстрый помощник

Добавлен скрипт:

- `scripts/git-branch-lanes.sh`

Команды:

- `scripts/git-branch-lanes.sh status`
- `scripts/git-branch-lanes.sh fetch`
- `scripts/git-branch-lanes.sh sync-main`
- `scripts/git-branch-lanes.sh sync-work`
- `scripts/git-branch-lanes.sh back-work`

Скрипт специально без разрушительных действий: только безопасный fetch/switch/pull `--ff-only`.

## 6. Правило именования новых веток

Если нужна новая рабочая ветка:

- использовать префикс `codex/`;
- имя в формате `codex/<что-делаем>-<дата-или-короткий-контекст>`.

Пример:

- `codex/contracts-mobile-drift-20260319`

## 7. Когда закрывать ветку

Ветку можно убирать из активных, когда:

- задача завершена и слита;
- документация обновлена;
- на ней нет незавершенных обязательств.

До этого момента ветка считается рабочей, а не музейной.
