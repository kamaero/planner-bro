## P0 (ближайшие релизы)

1. Роли и права 2.0 (тонкая модель)
   - Отдельные права: удаление, импорт, управление командой, массовые операции.
   - Модель прав должна работать одинаково для web/PWA/mobile.

2. История изменений задачи (audit log)
   - Кто и что поменял: статус/срок/исполнителя/описание/приоритет/прогресс.
   - Удобная лента в карточке задачи для разборов.

3. Массовые операции по задачам (bulk edit 2.0)
   - Массовое назначение/сроки/приоритет/теги.
   - Перенос между проектами (с проверкой прав и сохранением связей/истории).

4. Основание запуска проекта
   - Поле: номер приказа/протокола с датой.
   - Возможность прикрепить файл-основание.
   - Отображение меткой в проекте (и рядом с прогрессом/в Gantt-представлении).

5. Контроль СКИ
   - Чекбокс рядом с приоритетом проекта/задачи.
   - Если включен — приоритет принудительно максимальный.

6. Шаблоны проектов и чек-листов под ИТ-отдел
   - Типовые проекты: “закупка”, “внедрение”, “интеграция”.
   - Создание стандартизированного проекта в 1–2 клика.

7. Расписание/календарь загрузки команды
   - Перегруз/окна/конфликты дедлайнов.
   - Фильтры по роли/команде/проекту.

8. Мобильный MVP (PWA сначала, потом native)
   - Быстрые апдейты статуса/комменты/уведомления без долгой нативной разработки.

9. AI-помощник по проекту (на базе существующего AI pipeline)
   - Еженедельные сводки: “что сдвинулось, что стоит, где риски”.
   - Ссылки на конкретные задачи/блокеры.

10. Excalidraw-встраивание в карточку проекта
   - Доска брейншторма + привязка заметок/решений к задачам.

## P1 (устойчивость и эксплуатация)

12. Нотификации (единый контракт событий)
   - Унифицировать типы событий backend/web/mobile.
