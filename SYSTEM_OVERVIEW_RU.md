# PlannerBro: краткая справка

## Что это

**PlannerBro** — система управления ИТ-проектами и задачами для рабочих команд и руководителей.

Это не просто трекер задач, а единая операционная панель департамента: проекты, задачи, сроки, ответственность, эскалации, уведомления и контроль исполнения.

## Зачем нужна

- Чтобы менеджеры и исполнители **ежедневно держали статус задач актуальным**.
- Чтобы руководитель видел реальную картину по проектам, а не “ручные отчёты по запросу”.
- Чтобы не терялись сроки, назначения и зоны ответственности.
- Чтобы важные события происходили “под контролем”: кто назначен, кто онлайн, какие уведомления ушли, где есть риски.

## Ключевые возможности

- Дэшборд IT с группировкой проектов по отделам.
- Проекты и задачи: статусы, прогресс, приоритеты, дедлайны, комментарии.
- LIST и Gantt представления, сортировка и массовые операции.
- Связи между задачами (зависимости) и критический путь.
- Полноценная “Команда” с оргструктурой департамента и управлением ролями/правами.
- Уведомления:
  - in-app и realtime,
  - email c deep-link в нужный проект/задачу,
  - Telegram-бот со сводками по расписанию и командами в группе.
- Напоминания по статусам с гибкой логикой check-in.
- Правило для задач “Контроль СКИ”: ежедневные напоминания за 5 дней до дедлайна.
- Ежедневный аудит объектов без менеджера/админа.
- Шифрованное корпоративное хранилище документов.
- Мониторинг фоновой активности в интерфейсе (например, отправка email-уведомлений).

## Для кого

- Руководитель ИТ-департамента.
- Заместители, ГИПы, начальники отделов.
- Менеджеры проектов.
- Исполнители и команды сопровождения/внедрения.

## Технологии (кратко)

- Backend: FastAPI + PostgreSQL + Redis + Celery.
- Frontend: React + TypeScript.
- Realtime: WebSocket.
- Уведомления: SMTP, Firebase Push, Telegram Bot.
- Инфраструктура: Docker + Nginx.

## Практический эффект

- Рост дисциплины обновления статусов.
- Снижение управленческой “слепой зоны”.
- Быстрое выявление просрочек и блокеров.
- Прозрачная ответственность по проектам и задачам.
- Быстрая масштабируемость на несколько отделов и уровней подчинения.
